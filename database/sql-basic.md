# sql-basic

## 개요
SQL(Structured Query Language)은 관계형 데이터베이스(RDBMS)에서 데이터를 정의하고, 조회하고, 수정하고, 권한을 관리하기 위한 언어입니다.

국제 표준이 존재하지만, DBMS 제품마다 표준을 기반으로 약간의 차이가 있을 수 있습니다.

## 정의
- SQL: RDBMS에서 데이터를 조작/관리하기 위한 언어
- DDL(Data Definition Language): 데이터 구조(테이블/스키마)를 정의하는 언어
- DML(Data Manipulation Language): 데이터를 조회/삽입/수정/삭제하는 언어
- DCL(Data Control Language): 권한(접근 제어)을 관리하는 언어

## 핵심 개념
### 1) SQL은 “표준이 있지만 DBMS마다 차이가 있다”
- 국제 표준이 존재합니다.
- 하지만 MySQL, PostgreSQL, Oracle 같은 제품이 각각 구현하고 확장하면서 문법/기능에 차이가 생길 수 있습니다.
- 실무에서는 “표준 SQL + DBMS별 확장 문법”을 같이 접하게 됩니다.

### 2) DDL, DML, DCL로 역할을 나눠서 이해하면 쉽다
- DDL: 테이블 같은 구조를 만든다/바꾼다/삭제한다
- DML: 테이블 안의 데이터를 다룬다
- DCL: 누가 데이터를 볼 수 있는지 권한을 준다/회수한다

## DDL (Data Definition Language)
### 목적
- 테이블이나 관계의 구조를 생성/수정/삭제합니다.

### 대표 명령
- CREATE: 데이터베이스/테이블 생성
- ALTER: 테이블 구조 변경(컬럼 추가/삭제/수정 등)
- DROP: 데이터베이스/테이블 삭제(구조 자체 삭제)
- TRUNCATE: 테이블의 “데이터만” 비우고 구조는 유지

### 실무 포인트
- DROP은 구조 자체를 날립니다. 위험도가 큽니다.
- TRUNCATE는 데이터를 전체 삭제하는 느낌이라 빠르지만 롤백/제약에서 주의가 필요합니다.
- 운영 DB에서는 DDL 변경이 장애로 이어질 수 있어서, 변경 절차(점검/배포/백업)를 중요하게 다룹니다.

## DCL (Data Control Language)
### 목적
- 사용자 또는 역할(ROLE)에 권한을 부여/회수합니다.

### 대표 명령
- GRANT: 권한 부여
- REVOKE: 권한 회수

### 실무 포인트
- 운영에서는 “최소 권한 원칙”이 기본입니다.
- 애플리케이션 계정에 필요한 권한만 주고, 불필요한 권한은 제거합니다.

## DML (Data Manipulation Language)
### 목적
- 테이블의 데이터를 조회/삽입/수정/삭제합니다.

### 대표 명령
- SELECT: 조회
- INSERT: 삽입
- UPDATE: 수정
- DELETE: 삭제

### 실무 포인트
- SELECT는 가장 자주 쓰는 기본입니다.
- UPDATE/DELETE는 WHERE 조건 실수로 대량 변경/삭제 사고가 나기 쉬워서 주의합니다.

## 전체 흐름(개념)
1) DDL로 테이블 구조를 만든다
2) DML로 데이터를 넣고(INSERT), 조회하고(SELECT), 수정하고(UPDATE), 삭제한다(DELETE)
3) DCL로 사용자/서비스 계정이 접근할 수 있는 권한을 통제한다

## 용어 정리
| 용어 | 뜻 |
| --- | --- |
| SQL | RDBMS에서 데이터를 다루는 언어 |
| DDL | 구조 정의(테이블 생성/변경/삭제) |
| DML | 데이터 조작(조회/삽입/수정/삭제) |
| DCL | 권한 제어(부여/회수) |
| CREATE | 생성 |
| ALTER | 변경 |
| DROP | 구조 삭제 |
| TRUNCATE | 데이터 초기화(구조 유지) |
| GRANT | 권한 부여 |
| REVOKE | 권한 회수 |

- 권한(DCL)을 잘못 주면 어떤 보안 문제가 생길까?

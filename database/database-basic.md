# database-basic

## 개요
Database(DB)는 “데이터의 집합”입니다.
서비스에서 발생하는 정보(메시지, 사진, 주문, 결제 등)를 저장하고, 조회/수정/삭제할 수 있게 관리합니다.

## 정의
- DB(Database): 데이터를 저장하는 공간(데이터의 집합)
- DBMS(Database Management System): DB를 관리/운영하는 소프트웨어
- RDBMS(Relational DBMS): 관계형 DBMS, 데이터를 테이블(행/열) 구조로 저장하는 DBMS
- Table: 데이터 저장의 기본 단위
- Column: 테이블의 열, 데이터의 속성(예: 이름, 전화번호)
- Row: 테이블의 행, 실제 데이터 한 건(레코드)
- FK(Foreign Key): 다른 테이블의 데이터와 연결(참조)하기 위한 키

## 핵심 개념
### 1) RDBMS는 “테이블 구조”로 데이터를 저장한다
- 테이블은 column과 row로 구성됩니다.
- row는 한 사람/한 주문 같은 “데이터 1건”을 의미합니다.
- column은 그 데이터의 “속성”을 의미합니다.

### 2) FK로 테이블끼리 관계를 만든다
- RDBMS의 강점 중 하나가 테이블 간 관계를 만들고,
  이를 기반으로 데이터를 조합해서 조회할 수 있다는 점입니다.
- 예: 주문 테이블의 user_id(FK)가 유저 테이블 id(PK)를 참조해서
  “이 주문은 누가 했는지”를 연결할 수 있습니다.

### 3) RDBMS 제품은 다양하지만 큰 틀은 비슷하다
- MySQL, PostgreSQL, Oracle 등이 대표적입니다.
- 강의에서는 MySQL을 사용한다고 안내합니다.

## 동작 방식(큰 그림)
1) 애플리케이션이 DBMS에 쿼리(SQL)를 보냅니다.
2) DBMS가 디스크/메모리에서 데이터를 찾아 처리합니다.
3) 결과를 애플리케이션에 반환합니다.
4) 애플리케이션은 그 결과를 가공해서 응답(JSON 등)으로 반환합니다.

## 실무 포인트
- “DB”와 “DBMS”를 구분해서 말할 수 있어야 합니다.
  - DB는 데이터 자체/저장소 의미
  - DBMS는 DB를 운영하는 프로그램(MySQL, PostgreSQL 같은 것)
- RDBMS에서 중요한 이유
  - 데이터 무결성(관계/FK로 이상한 값이 들어가는 걸 막기 쉬움)
  - 관계 기반 조회(조인 등)
- MySQL vs PostgreSQL은 둘 다 많이 쓰고, 팀/환경/요구사항에 따라 선택합니다.
  - 강의에서는 MySQL을 쓸 예정이고, 배포 시 AWS RDS 같은 관리형 DB를 붙일 수 있다고 안내합니다.

## 용어 정리
| 용어 | 뜻 |
| --- | --- |
| DB | 데이터의 집합 |
| DBMS | DB를 관리/운영하는 소프트웨어 |
| RDBMS | 테이블 기반 관계형 DBMS |
| Table | 데이터 저장 단위 |
| Column | 속성(열) |
| Row | 데이터 1건(행) |
| FK | 다른 테이블을 참조하는 키 |

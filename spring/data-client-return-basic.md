# data-client-return-basic

## 개요
요즘 웹에서는 프론트엔드와 백엔드가 분리되는 경우가 많아서,
서버는 화면(html/css/js)을 직접 만들기보다 “요청에 필요한 데이터”를 JSON으로 내려주는 방식이 흔합니다.

## 정의
- View 응답: 서버가 html을 렌더링해서 반환하는 방식(템플릿 엔진 기반)
- Data 응답(JSON): 서버가 데이터(주로 JSON)를 반환하고, 브라우저가 화면을 조작하는 방식
- @ResponseBody: 컨트롤러 반환값을 “뷰 이름”이 아니라 “HTTP 응답 바디”로 사용
- @RestController: 컨트롤러 클래스 전체가 기본적으로 “응답 바디 반환” 모드가 되도록 하는 애너테이션
- Content-Type: 응답이 html인지(json인지)를 나타내는 헤더 값

## 핵심 개념
### 1) 응답 트렌드 변화
- 과거에는 서버가 화면을 직접 반환하는 구조가 많았습니다.
- 최근에는 프론트/백이 분리되면서 서버는 데이터(JSON)를 주고,
  프론트가 그 데이터를 이용해 화면을 바꾸는 구조가 일반적입니다.

### 2) SpringBoot에서 문자열 반환의 기본 해석
- 템플릿 엔진이 적용된 SpringBoot에서는 컨트롤러가 문자열을 반환하면
  templates 폴더에서 “해당 이름의 html”을 찾는 방향으로 동작할 수 있습니다.
- 그래서 화면이 아니라 데이터를 내려주려면 “뷰 렌더링 모드”를 꺼야 합니다.

### 3) JSON을 내려주는 스위치
- @ResponseBody가 붙으면 반환값은 “응답 바디”로 내려갑니다.
- @RestController를 쓰면 클래스 내 메서드들이 기본적으로 같은 효과를 가집니다.

### 4) 객체를 반환하면 JSON으로 변환된다
- 컨트롤러가 자바 객체(DTO)를 반환하면 Spring이 JSON으로 변환해서 내려줍니다.
- 이때 응답의 Content-Type은 보통 application/json 형태가 됩니다.

## 동작 방식
1) 클라이언트가 요청을 보냄(페이지 요청 또는 AJAX/fetch 같은 데이터 요청)
2) 컨트롤러 메서드 실행
3) 반환값 처리
- @ResponseBody(또는 @RestController) 있으면 “응답 바디”로 내려감
- 없으면 “뷰 이름”으로 해석되어 템플릿 렌더링 흐름으로 갈 수 있음
4) 응답 헤더(Content-Type)가 상황에 따라 달라짐
- html 응답이면 text/html
- JSON 응답이면 application/json

## 실무 포인트
- API 서버는 보통 @RestController를 기본으로 사용합니다.
- JSON을 문자열로 직접 만들기보다는 DTO를 반환하는 방식이 안전하고 유지보수에 좋습니다.
- 클라이언트에서 JSON을 기대했는데 화면(html)이 내려오면,
  “@ResponseBody/@RestController 누락” 또는 “뷰 렌더링 경로로 탄 것”을 먼저 의심하면 됩니다.
- Content-Type만 봐도 지금 응답이 html인지 json인지 빠르게 판단할 수 있습니다.

## 용어 정리
| 용어 | 뜻 |
| --- | --- |
| View 렌더링 | 서버가 html을 만들어 반환 |
| JSON 응답 | 서버가 데이터를 반환하고 프론트가 화면 반영 |
| @ResponseBody | 반환값을 HTTP 바디로 내려줌 |
| @RestController | 클래스 전체에 응답 바디 반환 효과 |
| Content-Type | 응답 데이터 타입 헤더 |
